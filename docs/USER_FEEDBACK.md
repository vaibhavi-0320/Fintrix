# User Feedback and Iteration Log

## Objective

Capture real-user product feedback and ensure at least one meaningful iteration is implemented and documented.

## Feedback Collection Channels

- In-session usability observations during MVP walkthroughs
- Feature comments from simulation and marketplace flows
- Assistant usage behavior and support questions

## Key Feedback Themes

1. Marketplace initially appeared empty and reduced user trust.
2. Users wanted clearer pre-decision risk guidance before funding.
3. Mobile navigation needed a more reliable interaction model.
4. Product naming and brand consistency needed cleanup.

## Implemented Iterations

### Iteration 1: Marketplace usability fix (completed)
- Added fallback mock opportunities when backend data is empty.
- Added richer filters (status, risk, APY, duration).
- Added improved card UI and empty-state guidance.
- **Impact:** users can evaluate opportunities immediately without blank states.

### Iteration 2: AI decision support (completed)
- Added Risk Detector prompts before decision flows.
- Added assistant quick prompts for red-flag checks.
- **Impact:** improved confidence and reduced premature action patterns.

### Iteration 3: Mobile UX pass (completed)
- Added mobile bottom navigation and spacing adjustments.
- Improved responsive behavior across simulation and marketplace surfaces.
- **Impact:** smoother mobile task completion path.

## Current Status

- ✅ Feedback documented
- ✅ At least one iteration shipped
- ✅ Iteration outcomes reflected in product behavior
